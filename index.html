Git commands

git --version

check is there git on your computer and find out it verion

git config --global --list

find out settings of your GLOBAL settings for your git on your current computer

if you have problems with rights, run this commandgit config --global --add safe.directory path (for example E:/OpenServer4/domains/test1)

git config --global user.name

set your NAME for your git global setting on your computer

git config --global user.email

set your EMAIL for your git global setting on your computer

git config --global --list

will show information about the global settings of the current local repository

git init

turn current folder into repository and monitor the changes that occur there

git remote add origin { url repository }

create a connection between a local repository and a remote one by URL

git clone { url repository }

download all files of a remote repository and create a local copy of this (remote) repository

git status

show information about all uncommitted files in the current local repository

git add /some_files

prepares these files for the next commit by adding them to the git index

git add .

prepares all files for the next commit by adding them to the git index

git commit -m "name_commit"

creates a commit from the git index and assigns it a unique code (hash). you can set the commit name with the -m parameter

git log

will display information about all commits in chronological order

git log -1

will only show information about the latest commit in chronological order

git checkout { hash }

revert (return all project files) to a commit specifying the hash

git push { name remote repository (for example origin) } { branch_name (for example master) }

publishes commits from the current local repository to the remote repository

git pull { name remote repository (for example origin) } { branch (for example master) }

downloads from a given remote repository all commits that are not yet in the current local repository

git branch

will show a list of all branches available in the current local repository

git branch { branch_name }

creates a branch with the specified branch name

git checkout { branch_name }

switch to another branch with the specified branch name

git merge { branch_name }

merges the specified branch into the current branch we are on right now with one large commit

git rebase { branch_name }

merges with the specified branch into the current branch on which we are right now with the history of all commits that were made on the branch from which the current branch is merged
